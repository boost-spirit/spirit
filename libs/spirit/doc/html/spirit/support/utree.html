utree

utree is a hierarchical, dynamic data structure that can represent abstract syntax trees. utree can store the following: 

struct utree_type
{
    enum info
    {
        nil_type, 
        list_type, 
        range_type, 
        reference_type, 
        any_type, 
        function_type,  
        
        // numeric atoms
        bool_type,
        int_type,
        double_type,

        // text atoms
        string_type, 
        string_range_type,
        symbol_type, 
        binary_type 

    };
};

nil is the sentinel utree type.

A doubly linked list of utrees.

A range of list::iterators.

A reference to another utree.

A pointer or reference to any C++ type. RTTI is used to retrieve the type dynamically.

A utree holding a stored_function<F> object, where F is an unary function object taking a scope as it's parameter and returning a utree.

utree strings store utf8 data.

utree symbols also use utf8 encoding.

This utree type can store binary data.

Note

utree has minimal memory footprint. The data structure size is 16 bytes on a 32-bit platform. Being a container of itself, it can represent tree structures.

      static visit(utree const&, utree&, F);

    template <class F>
      typename boost::result_of<F(utree&, utree&)>::type
      static visit(utree&, utree&, F);

    ////////////////////////////////////////////////////////////////////////

      void push_back(T const&);
    template <class T>
      void push_front(T const&);
    template <class T>
      iterator insert(iterator, T const&);
    template <class T>
      void insert(iterator, std::size_t, T const&);
    template <class Iter>
      void insert(iterator, Iter, Iter);

    // erasure
    void pop_front();
    void pop_back();

Function Object Interface

The stored_function template class can to store a unary function objects with a signature of utree(scope const&) as a utree node. 

struct function_base
{
    virtual ~function_base() {};
    virtual utree operator()(scope const& env) const = 0;
    virtual function_base* clone() const = 0; 
};

template <typename F>
struct stored_function : function_base
{
    F f;
    stored_function(F f = F());
    virtual ~stored_function();
    virtual utree operator()(scope const& env) const;
    virtual function_base* clone() const;
};

Calling f.clone() must return a newly allocated function_base instance ff that is equal to f.

Exceptions 

struct utree_exception : std::exception {}; 
  

struct bad_type_exception : utree_exception 
{
    virtual const char* what() const throw()
    {
        return "utree: Illegal operation for currently stored data.";
    }
};

All exceptions thrown by utree are derived from utree_exception.

bad_type_exception is thrown whenever somebody calls a member function which applies to certain stored utree_type's only, but this precondition is violated as the utree instance holds some other type.
