Compound Attribute Rules

Notation

The notation will be for instance: 

a: A, b: B --> (a << b): tuple<A, B>

which reads as: given, a and b are generators, and A is the type of the attribute of a, and B is the type of the attribute of b, then the type of the attribute of a << b will be tuple<A, B>.

Table 5. Spirit.Karma compound generator attribute types

Expression

Attribute

sequence (<<) 

a: A, b: B --> (a << b): tuple<A, B>
a: A, b: Unused --> (a << b): A
a: Unused, b: B --> (a << b): B
a: Unused, b: Unused --> (a << b): Unused

a: A, b: A --> (a << b): vector<A>
a: vector<A>, b: A --> (a << b): vector<A>
a: A, b: vector<A> --> (a << b): vector<A>
a: vector<A>, b: vector<A> --> (a << b): vector<A>

alternative (|) 

a: A, b: B --> (a | b): variant<A, B>
a: A, b: Unused --> (a | b): A
a: Unused, b: B --> (a | b): B
a: Unused, b: Unused --> (a | b): Unused
a: A, b: A --> (a | b): A

kleene (unary *) 

a: A --> *a: vector<A>
a: Unused --> *a: Unused

plus (unary +) 

a: A --> +a: vector<A>
a: Unused --> +a: Unused

list (%) 

a: A, b: B --> (a % b): vector<A>
a: Unused, b: B --> (a % b): Unused

repetition 

a: A --> repeat(...,...)[a]: vector<A>
a: Unused --> repeat(...,...)[a]: Unused

optional (unary -) 

a: A --> -a: optional<A>
a: Unused --> -a: Unused

and predicate (unary &)

a: A --> &a: A

not predicate (unary !)

a: A --> !a: A
